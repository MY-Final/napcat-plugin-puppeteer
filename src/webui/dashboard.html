核心服务

POST /puppeteer/screenshot

通用截图接口，支持 URL、本地文件路径或直接传入 HTML 字符串进行渲染。

Request Body (JSON)
参数名 类型 必填 说明
file string 是 目标内容 (URL / HTML代码 / 文件路径)
file_type string 否 指定内容类型: url | htmlString | file | auto(默认)
selector string 否 CSS 选择器，只截取指定元素。默认 body
omitBackground boolean 否 是否隐藏默认背景（设为透明）。默认 false
data object 否 Handlebars 模板数据，仅当 file 为模板时有效
waitSelector string 否 等待该元素出现后再截图
setViewport object 否 { width, height, deviceScaleFactor }
Response JSON
{
  "code": 0,
  "data": "Base64String...",  // 图片数据
  "message": "OK",
  "time": 150                 // 耗时(ms)
}
POST /puppeteer/render

/screenshot 的语义化别名，专门用于 HTML 模板渲染。

参数名 类型 必填 说明
html string 是 HTML 模板字符串
data object 否 模板插值数据
GET /puppeteer/screenshot

轻量级 URL 截图接口，适合快速调试或简单场景。

Query Parameters
参数名 必填 说明
url 是 目标网页地址
width 否 视口宽度 (默认 1280)
height 否 视口高度 (默认 800)
selector 否 元素选择器
raw 否 如果为 true，直接返回 image/png 流，不包装 JSON

浏览器控制

GET /puppeteer/browser/status

获取 Puppeteer 实例的详细状态。

{
  "code": 0,
  "data": {
    "connected": true,      // 浏览器是否连接
    "version": "Chrome...", // 版本信息
    "pageCount": 1,         // 打开的页面数
    "pid": 12345,           // 进程 ID
    "executablePath": "..." // 浏览器路径
  }
}
POST /puppeteer/browser/{action}

生命周期控制接口，支持 start, stop, restart。

POST /puppeteer/browser/start POST /puppeteer/browser/stop POST /puppeteer/browser/restart 
{ "code": 0, "message": "Browser started successfully" }

系统配置

POST /puppeteer/config

热更新插件配置，实时生效（部分浏览器配置需重启生效）。

Request Body
{
  "browser": {
    "headless": true,
    "args": ["--no-sandbox"]
  },
  "maxPages": 10,
  "lockTimeout": 30000
}
GET /puppeteer/status

获取插件整体运行统计。

{
  "totalRenders": 100,
  "failedRenders": 2,
  "uptimeFormatted": "2小时 15分"
}

调用示例

其他插件如何调用渲染 API

fetch('/api/Plugin/ext/插件名/puppeteer/render', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    html: '<h1>Hello {{name}}</h1>',
    data: { name: 'World' },
    encoding: 'base64'
  })
})
fetch('/api/Plugin/ext/插件名/puppeteer/screenshot', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    file: 'https://example.com',
    encoding: 'base64'
  })
})
